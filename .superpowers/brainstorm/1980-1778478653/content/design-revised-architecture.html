Revised Architecture: 5-Layer Design

Merging document's layering with our Actor Network + Event Bus approach, powered by XState

5-Layer Architecture (Revised)
┌─────────────────────────────────────────────────────────────┐
│  L1  Trigger Layer                                          │
│      UserAction (kubectl apply/delete/scale)                 │
│      TickClock (simulated time: probes, retries, heartbeats) │
└──────────────────────────┬──────────────────────────────────┘
                           │ dispatch(event)
┌──────────────────────────▼──────────────────────────────────┐
│  L2  MessageBus Layer                                        │
│      MessageBus (publish / route / subscribe)                │
│      EventHistory (record all events, deterministic replay)  │
│      PlaybackController (controls delivery timing)           │
└──────────────────────────┬──────────────────────────────────┘
                           │ route → Actor.mailbox
┌──────────────────────────▼──────────────────────────────────┐
│  L3  Actor Runtime Layer                                     │
│      ActorRegistry (lookup by ID, multi-instance support)    │
│      Actor<S, E> (base class: mailbox + lifecycle)           │
│      XState Machine (FSM engine per actor)                   │
│      TransitionDSL (declarative rules with guards)           │
└──────────────────────────┬──────────────────────────────────┘
                           │ side-effects / read-write
┌──────────────────────────▼──────────────────────────────────┐
│  L4  K8s Domain Layer                                       │
│      APIServerActor · SchedulerActor · KubeletActor           │
│      ControllerManagerActor · OperatorActors                  │
│      EtcdStore (in-memory resource storage, versioned)        │
│      PodObject · NodeObject (pure data)                       │
└──────────────────────────┬──────────────────────────────────┘
                           │ state changes → notify
┌──────────────────────────▼──────────────────────────────────┐
│  L5  View Bridge Layer                                       │
│      ReactiveStore (adapts actor state to React)             │
│      useSyncExternalStore (React 18 standard API)            │
│      Zustand (existing UI state: speed, panel tabs, etc.)    │
│      FlowCanvas · Timeline · DetailPanel · Toolbar            │
└─────────────────────────────────────────────────────────────┘

Layer Dependencies (unidirectional, downward only)

L1 Trigger

Knows: L2 (dispatches to bus)
Does NOT know: L3, L4, L5

L2 MessageBus

Knows: L3 (routes to actors)
Does NOT know: L1, L4, L5

L3 Actor Runtime

Knows: XState FSM engine only
Does NOT know: L1, L4, L5

L4 K8s Domain

Knows: L2 (bus), L3 (Actor base)
Does NOT know: L1, L5

L5 View Bridge

Knows: L4 (subscribes to state)
Does NOT know: L1, L2, L3

Key Design Invariants

  • FSM engine is pluggable — L3 defines the Actor interface, XState is one implementation. Can swap to custom FSMEngine later.
  • EtcdStore separates concerns — K8s resources live in EtcdStore (L4), not in Zustand. Zustand only holds UI state.
  • Deterministic replay — same event sequence → same state. EventHistory enables time-travel debugging.
  • TickClock decouples time — no real timers in actors. All time-based behavior driven by TICK events.
