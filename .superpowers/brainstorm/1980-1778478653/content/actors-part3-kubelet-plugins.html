Actor FSM Definitions — Part 3: Kubelet & Plugin Actors

KubeletActor (per node) + CRIActor, CNIActor, CSIActor

KubeletActor (per node)
States:
  idle · creating_sandbox · settingup_network
  mounting_volumes · pulling_image · starting_container
  running · restarting · terminating · error

Happy Path:

idle ──[POD_BOUND]──→ creating_sandbox
  │ route CREATE_SANDBOX → CRI
  │
creating_sandbox ──[SANDBOX_OK]──→ settingup_network
  │ route CNI_SETUP → CNI       │
creating_sandbox ──[SANDBOX_FAIL]──→ error
  │
settingup_network ──[CNI_OK]──→ mounting_volumes
  │ route CSI_STAGE + CSI_PUBLISH │
settingup_network ──[CNI_FAIL]──→ error
  │
mounting_volumes ──[CSI_OK]──→ pulling_image
  │ route PULL_IMAGE → CRI       │
mounting_volumes ──[CSI_FAIL]──→ error
  │
pulling_image ──[PULL_OK]──→ starting_container
  │ route START_CONTAINER → CRI  │
pulling_image ──[PULL_FAIL]──→ error
  │
starting_container ──[START_OK]──→ running
  │ publish POD_RUNNING           │
  │ update pod status in etcd     │
starting_container ──[START_FAIL]──→ error

Running Phase:
running ──[TICK]──→ running
  │ run health probe
  │ if fail → emit PROBE_FAIL to self
  │
running ──[PROBE_FAIL]──→ restarting
  │ restart container             │
restarting ──[TICK]──→ running
  │ (or → error if retry exceeded)
  │
running ──[POD_DELETED]──→ terminating
  │ cleanup CNI, CSI, CRI        │
terminating ──→ idle

Error Recovery:
error ──[TICK]──→ idle
  │ publish POD_FAILED
  │ backoff, controller will reconcile
CRIActor / CNIActor / CSIActor
All three share the same FSM shape:

idle ──[<OPERATION>]──→ processing
  │ execute operation
  │ emit <OPERATION>_OK or
  │      <OPERATION>_FAIL
  ▼
idle ←── processing

CRIActor Operations:
  CREATE_SANDBOX → SANDBOX_OK / SANDBOX_FAIL
  PULL_IMAGE     → PULL_OK / PULL_FAIL
  START_CONTAINER → START_OK / START_FAIL

CNIActor Operations:
  CNI_SETUP → CNI_OK / CNI_FAIL

CSIActor Operations:
  CSI_STAGE_VOLUME   → CSI_STAGE_OK / CSI_STAGE_FAIL
  CSI_PUBLISH_VOLUME → CSI_PUBLISH_OK / CSI_PUBLISH_FAIL

Fault Injection:
Scenario errors are injected by
publishing *_FAIL instead of *_OK
when the scenario config matches
the current operation type.

Design Notes

Per-node Kubelet — each node gets its own KubeletActor instance (actorId: kubelet:node-1), supporting multi-node simulation.

Plugin Actors are thin — they model external systems (CRI/CNI/CSI) as simple request/response FSMs. Fault injection works by overriding the response event.

Cleanup on terminate — terminating state sends cleanup requests to CNI/CSI/CRI in reverse order before returning to idle.
