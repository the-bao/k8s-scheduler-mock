How actors collaborate through Event Bus + how playback works

Full Pod Lifecycle — Event Flow
── Phase 1: Submit ──────────────────────────────────────────

UserAction ──── USER_APPLY ────→ [API-Server] idle → validating → storing
                                  │ WRITE_REQUEST
                                  ▼
                                [Etcd] idle → writing → idle
                                  │ WATCH_EVENT (key prefix: /registry/pods/*)
                                  ▼
                                [API-Server] → broadcasting → idle
                                  │ CREATE_POD_RESPONSE → User
                                  │ POD_CREATED (broadcast)

── Phase 2: Controller ─────────────────────────────────────

                                [Controller-Mgr] watching → reconciling
                                  │ Watch triggered by etcd event
                                  │ For non-Pod resources: BFS chain reconcile
                                  │   Deployment → ReplicaSet → Pod
                                  │ For Pod: check if operator plugins apply
                                  │ If diff > 0 → updating → watching
                                  │ publish CREATE_POD per diff action

── Phase 3: Scheduling ─────────────────────────────────────

                                [Scheduler] idle → filtering
                                  │ Subscribe to POD_PENDING
                                  │ Filter nodes by resource fit
                                  │ candidates > 0 → scoring → binding
                                  │
                                  │ BIND_REQUEST
                                  ▼
                                [API-Server] idle → updating → idle
                                  │ Write pod.nodeName to etcd
                                  │
                                  ▼
                                [Etcd] idle → writing → idle
                                  │ WATCH_EVENT → Kubelet (node match)

── Phase 4: Kubelet ────────────────────────────────────────

                                [Kubelet:node-1] idle → creating_sandbox
                                  │ CREATE_SANDBOX → [CRI]
                                  ▼
                                settingup_network
                                  │ CNI_SETUP → [CNI]
                                  ▼
                                mounting_volumes
                                  │ CSI_STAGE + CSI_PUBLISH → [CSI]
                                  ▼
                                pulling_image
                                  │ PULL_IMAGE → [CRI]
                                  ▼
                                starting_container
                                  │ START_CONTAINER → [CRI]
                                  ▼
                                running
                                  │ POD_RUNNING → API-Server → Etcd
                                  │
                                  ▼
                                [API-Server] UPDATE_STATUS → [Etcd]
PlaybackController FSM
States: idle · running · paused · completed · stepping

Transitions:

idle ──[PLAY]──→ running
idle ──[STEP_FORWARD]──→ stepping
idle ──[LOAD_SCENARIO]──→ idle
  │ reset all actors, load scenario config

running ──[TICK]──→ running
  │ deliver next buffered event from EventBus
  │ record snapshot for backward navigation
  │ if no more events → completed
  │
running ──[PAUSE]──→ paused
running ──[RESET]──→ idle

paused ──[PLAY]──→ running
paused ──[STEP_FORWARD]──→ stepping
paused ──[STEP_BACKWARD]──→ stepping
  │ restore previous snapshot
paused ──[JUMP_TO(n)]──→ stepping
  │ restore snapshot at index n
paused ──[RESET]──→ idle

stepping ──[DELIVERED]──→ paused
  │ one event delivered, pause again

completed ──[RESET]──→ idle
completed ──[JUMP_TO(n)]──→ stepping

Speed Control:
TICK interval = baseInterval / speed
  speed: 0.5x → 2s per tick
  speed: 1x   → 1s per tick
  speed: 2x   → 500ms per tick
  speed: 5x   → 200ms per tick

Snapshot Strategy:
  Snapshot = { all actor states + EtcdStore data + event log index }
  Taken after every event delivery
  Max snapshots: 500 (ring buffer, oldest discarded)
