Actor FSM Definitions — Part 1: Control Plane Core

APIServerActor & EtcdActor — the request handling and storage backbone

APIServerActor
States:
  idle · validating · storing ·
  broadcasting · updating

Transitions:

idle ──[USER_APPLY]──→ validating
  │ validate manifest
  │
validating ──[VALID_OK]──→ storing
  │                     │
validating ──[VALID_FAIL]──→ idle
  │ emit error to user   │
  │                      ▼
storing ──[WRITE_OK]──→ broadcasting
  │                    │
storing ──[WRITE_FAIL]──→ idle
  │ emit error          │ broadcast watch events
  │                     │ (POD_CREATED, POD_UPDATED)
  │                     ▼
                  idle ←── broadcasting

Also handles:
idle ──[BIND_REQUEST]──→ updating
  │ update pod.nodeName
  │ broadcast POD_BOUND
  ▼
idle ←── updating

idle ──[STATUS_UPDATE]──→ updating
  │ update pod.status
  │ broadcast POD_UPDATED
  ▼
idle ←── updating

Emitted Events

POD_CREATED → Scheduler, ControllerMgr
POD_BOUND → Kubelet (matched node)
POD_UPDATED → ControllerMgr, View
ERROR_RESPONSE → User (on validation/store fail)
EtcdActor
States:
  idle · writing · reading

Transitions:

idle ──[WRITE_REQUEST]──→ writing
  │ write to in-memory map
  │ increment revision
  │ emit WRITE_RESPONSE
  │ emit WATCH_EVENT for subscribers
  ▼
idle ←── writing

idle ──[READ_REQUEST]──→ reading
  │ lookup from map
  │ emit READ_RESPONSE
  ▼
idle ←── reading

Watch Mechanism:
When a key is written:
  1. Store value in memory Map
  2. Increment revision counter
  3. Publish WATCH_EVENT_{type}
     to all subscribers of that
     key prefix

Subscriptions:
  /registry/pods/*     → Scheduler
                        ControllerMgr
  /registry/pods/*/status → Kubelet

Key Design Notes

  • Versioned storage — every write increments a revision number, enabling WATCH to detect changes
  • Watch = pub/sub on key prefix — mirrors real etcd Watch semantics
  • Pure data store — no business logic, just CRUD + notifications
  • Error state not needed — in-memory store doesn't fail (real etcd could, simulated doesn't)
